# HomePage

**Screen Name:** `HomeScreen`

**Purpose:**

- Serves as the central landing hub and portfolio introduction.
- Showcases featured projects and recent learning insights.
- Provides navigation to detailed lists and contact information.

**Entry Point:**

- App Launch
- Back navigation from `ProjectListScreen` or `ArticleListScreen`

**Exit Point:**

- **Internal:** `ProjectListScreen`, `ArticleListScreen`, `ProjectDetailScreen`, `ArticleDetailScreen`.
- **External:** GitHub, LinkedIn, Medium, Mailto (Email).

**Access Level:** Public

**Additional Notes:**

- "Contact" section acts as the footer.
- Images are handled as local assets.

**Dynamic Content Inventory:**

1. **Hero Section**
    - `hero.title`
    - `hero.subtitle`
2. **Featured Projects (List)**
    - `project.title`
    - `project.typeIcon` (Material Symbol Name, e.g., 'smartphone', 'groups')
    - `project.description`
    - `project.coverImageAsset` (Local Asset Path)
    - `project.tags` (List: Flutter, Hive, Bloc, etc.)
    - `project.sourceUrl` (GitHub Link)
3. **Featured Learnings (List)**
    - `article.title`
    - `article.date`
    - `article.readTime`
    - `article.tags`
    - `article.coverImageAsset` (Local Asset Path)
    - `article.summary`
4. **Profile / About**
    - `profile.bio`
    - `profile.avatarAsset` (Local Asset Path)
5. **Expertise (List)**
    - `skill.title` (e.g., State Management)
    - `skill.description`
    - `skill.icon` (Material Symbol)

### âš¡ `init_loadHomeData` Specification (Lifecycle)

1. **Action Definition & Logic**
    - **Action ID:** `init_loadHomeData`
    - **Trigger:** `initState` (Screen Load)
    - **Prerequisites:** None
    - **Method:** `fetchHomeContent()`
    - **Raw Inputs:** None
    - **Raw Outputs:** `HomeData` object (containing `List<Project>`, `List<Article>`, `ProfileData`)
    - **Purpose:** To populate the Homepage with the "Featured" subsets of data and the user's profile information immediately upon launch.
2. **UI State Machine**
    - **Idle:** Empty/Skeleton UI.
    - **Loading:** Display Skeleton Loader (shimmer effect) on cards.
    - **Success:** Render Hero, Project Grid (3 items), Article Grid (3 items), and About section.
    - **Error:** Display a "Retry" button or use fallback offline data (Hive).
    - **Network Issue:** Attempt to load from local cache (Hive) if available.

### âš¡ `logic_openExternalLink` Specification (Logic)

1. **Action Definition & Logic**
    - **Action ID:** `logic_openExternalLink`
    - **Trigger:** Tap (Social Icons: GitHub, LinkedIn, Medium)
    - **Prerequisites:** Valid URL string.
    - **Method:** `launchUrl(String url)`
    - **Raw Inputs:** `url` (String)
    - **Raw Outputs:** Boolean (`true` if launched successfully)
    - **Purpose:** Opens the user's social profiles in the system's default browser.
2. **UI State Machine**
    - **Idle:** Social Icon visible.
    - **Loading:** System-handled (OS transition).
    - **Success:** Browser opens.
    - **Error:** Show Toast: "Could not launch URL."

---

### âš¡ `logic_sendEmail` Specification (Logic)

1. **Action Definition & Logic**
    - **Action ID:** `logic_sendEmail`
    - **Trigger:** Tap (FAB or "Get In Touch" button)
    - **Prerequisites:** Mail app installed on device.
    - **Method:** `launchUrl('mailto:contact@example.com')`
    - **Raw Inputs:** None
    - **Raw Outputs:** Boolean
    - **Purpose:** Initiates the OS email intent to compose a message to the portfolio owner.
2. **UI State Machine**
    - **Idle:** Button visible.
    - **Loading:** System-handled.
    - **Success:** Email app opens.
    - **Error:** Show Toast: "No email client found."

---

### âš¡ `logic_openRepo` Specification (Logic)

1. **Action Definition & Logic**
    - **Action ID:** `logic_openRepo`
    - **Trigger:** Tap (GitHub Icon inside Project Card)
    - **Prerequisites:** Valid source URL in project model.
    - **Method:** `launchUrl(project.sourceUrl)`

    - **Raw Outputs:** Boolean
    - **Purpose:** Directly opens the specific source code repository for a project.
2. **UI State Machine**
    - **Idle:** Icon visible.
    - **Loading:** System-handled.
    - **Success:** Browser opens.
    - **Error:** Show Toast: "Repository link invalid."
